# Kirino

**ゼロトラスト認証・認可**——RBACにとどまらない——をRustエコシステムにもたらす原則的なフレームワーク。

KirinoはANSI INCITS 359-2004 RBACを完全に実装します（RBAC0/1/2）。職務分離、基数制限、時間的制約、ロール階層。単に「誰が何をできるか」ではなく、「どのような条件下で」を問います。

その上に、信頼スコアリング、異常検出、DO-178Cに着想を得た自律レベル（L0–L4）を持つ動的認可レイヤーが重なります——アクセス決定は静的なYes/Noではなく、リスクに適応します。実世界の認可を、アプリケーションに直接埋め込めるコンポーザブルなRustモジュールへと蒸留しました。
